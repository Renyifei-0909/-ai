# 打包成桌面程序 - 一步一步操作指南

按下面步骤依次做，即可把项目打成可在电脑上安装的桌面程序（如 .dmg 或 .exe）。

---

## 第一步：确认项目目录

**你要做的：**

1. 用资源管理器或 Finder 打开你的项目文件夹（包含 `package.json`、`electron`、`backend` 等的那一层）。
2. 记下这个文件夹的路径（例如：`/Users/你的用户名/Documents/GitHub/-ai` 或 `D:\Projects\-ai`）。

**说明：** 后面所有“在项目根目录执行”的命令，都是指在这个文件夹里操作。

---

## 第二步：检查是否已安装 Node.js

**你要做的：**

1. 打开「终端」（macOS / Linux）或「命令提示符 / PowerShell」（Windows）。
2. 输入下面这一行（照抄，按回车）：
   ```bash
   node -v
   ```
3. 看结果：
   - **若出现类似 `v18.0.0` 或 `v20.x.x`**：说明已安装，进入第三步。
   - **若提示“找不到命令”或 “command not found”**：需要先安装 Node.js。
     - 打开 https://nodejs.org ，下载并安装 **LTS 版本**。
     - 安装完成后**关闭并重新打开终端**，再执行一次 `node -v` 确认。

---

## 第三步：检查是否已安装 npm

**你要做的：**

1. 在终端里输入（按回车）：
   ```bash
   npm -v
   ```
2. 看结果：
   - **若出现数字（如 `10.2.0`）**：说明 npm 可用，进入第四步。
   - **若提示找不到命令**：一般随 Node.js 一起安装，请确认 Node.js 已正确安装并重启终端后再试。

---

## 第四步：进入项目根目录

**你要做的：**

1. 在终端里用 `cd` 命令进入你在第一步记下的项目路径。  
   示例（把路径改成你自己的）：
   - macOS / Linux：
     ```bash
     cd /Users/你的用户名/Documents/GitHub/-ai
     ```
   - Windows（PowerShell）：
     ```powershell
     cd D:\Projects\-ai
     ```
2. 确认当前在项目里：输入
   ```bash
   dir
   ```
   （Windows）或
   ```bash
   ls
   ```
   （macOS / Linux），应能看到 `package.json`、`electron`、`backend` 等。

---

## 第五步：安装前端与 Electron 依赖

**你要做的：**

1. 确保终端当前目录是项目根目录（上一步的 `cd` 已执行）。
2. 输入并回车：
   ```bash
   npm install
   ```
3. 等待执行结束（可能 1～3 分钟）：
   - **成功**：最后无报错，可能出现 `added xxx packages` 之类提示。→ 进入第六步。
   - **失败**：若出现红色报错，可尝试：
     - 检查网络是否正常；
     - 再执行一次 `npm install`；
     - 或使用国内镜像：`npm install --registry=https://registry.npmmirror.com`

---

## 第六步：检查是否已安装 Python

**你要做的：**

1. 在终端输入（二选一，先试第一个）：
   ```bash
   python3 --version
   ```
   若没有，再试：
   ```bash
   python --version
   ```
2. 看结果：
   - **若出现类似 `Python 3.10.x` 或更高**：说明已安装，进入第七步。
   - **若提示找不到命令**：需要先安装 Python。
     - 打开 https://www.python.org/downloads/ 下载并安装 **Python 3.10 或更高**。
     - 安装时勾选 “Add Python to PATH”（若安装程序有该选项）。
     - 安装后重新打开终端，再执行上述命令确认。

---

## 第七步：安装 Python 后端依赖

**你要做的：**

1. 确保当前仍在项目根目录。
2. 输入并回车：
   ```bash
   cd backend
   ```
3. 再输入并回车：
   ```bash
   pip install -r requirements.txt
   ```
   若系统提示用 `pip3`，则改为：
   ```bash
   pip3 install -r requirements.txt
   ```
4. 等待安装结束，无报错即可。
5. 回到项目根目录，输入并回车：
   ```bash
   cd ..
   ```
   （Windows 下同样是 `cd ..`。）

---

## 第八步：检查是否已安装 FFmpeg（可选但建议）

**你要做的：**

1. 在终端输入：
   ```bash
   ffmpeg -version
   ```
2. 若出现版本信息，说明已安装，可继续第九步。
3. 若未安装：
   - **macOS**：可执行 `brew install ffmpeg`（需先安装 Homebrew）。
   - **Windows**：从 https://ffmpeg.org 下载，解压后将 `ffmpeg.exe`、`ffprobe.exe` 所在目录加入系统 PATH。  
   打包本身不依赖本机 FFmpeg，但运行打包后的程序时，用户电脑需要 FFmpeg 才能使用视频相关功能。

---

## 第九步：执行打包命令

**你要做的：**

1. 确认终端当前在**项目根目录**（可再执行一次 `cd` 到你的项目路径，或执行 `cd ..` 直到回到有 `package.json` 的目录）。
2. 输入并回车：
   ```bash
   npm run electron:build
   ```
3. 等待完成（首次可能几分钟）：
   - 脚本会先把前端文件复制到 `dist/`，再调用 electron-builder 打包。
   - **成功**：最后无报错，终端可能出现 “packaging” 等提示。→ 进入第十步。
   - **失败**：若有报错，把终端里红色错误信息复制下来，便于排查（常见原因：路径不对、依赖未装全、权限问题）。

---

## 第十步：找到打包好的安装包

**你要做的：**

1. 在项目根目录下，打开名为 **`release`** 的文件夹。
2. 根据你的系统查看：
   - **macOS**：找 `AI智剪-0.1.0.dmg`（或类似名称的 .dmg 文件）；有时会在 `release/mac/` 下有 .app。
   - **Windows**：找 `AI智剪 Setup 0.1.0.exe`（或类似名称的 .exe 安装程序）。
3. 这个 .dmg 或 .exe 就是你需要的「桌面安装包」。

**说明：** 文件名里的「AI智剪」和版本号（如 0.1.0）来自 `package.json` 里的 `productName` 和 `version`，你改那里后重新打包就会变。

---

## 第十一步：在本机安装并运行（验证）

**你要做的：**

1. **macOS**：双击 .dmg，把应用拖到「应用程序」；在启动台或应用程序文件夹里打开「AI智剪」。
2. **Windows**：双击 .exe，按安装向导下一步完成安装；从开始菜单或桌面快捷方式打开「AI智剪」。
3. 若能正常打开界面，说明打包和安装都成功。

---

## 第十二步：把安装包发给别人用

**你要做的：**

1. 把 **`release`** 里的 .dmg（macOS）或 .exe（Windows）通过 U 盘、网盘、邮件等方式发给对方。
2. 告知对方：
   - 需要先安装再使用（双击安装包按提示操作）。
   - **运行本程序需要其电脑上已安装 Python 和 FFmpeg**（否则视频相关功能可能不可用）。若对方未安装，需先安装后再运行。

---

## 小结：你要做的步骤顺序

| 顺序 | 你要做的一件事 |
|------|----------------|
| 1 | 打开项目文件夹，记下路径 |
| 2 | 终端执行 `node -v`，没有就装 Node.js |
| 3 | 终端执行 `npm -v`，确认有 npm |
| 4 | 终端执行 `cd 你的项目路径`，进入项目根目录 |
| 5 | 执行 `npm install`，装完前端依赖 |
| 6 | 执行 `python3 --version` 或 `python --version`，没有就装 Python |
| 7 | 执行 `cd backend` → `pip install -r requirements.txt` → `cd ..` |
| 8 | （可选）执行 `ffmpeg -version`，没有就装 FFmpeg |
| 9 | 在项目根目录执行 `npm run electron:build` |
| 10 | 到项目里的 `release` 文件夹找 .dmg 或 .exe |
| 11 | 本机双击安装并打开，验证能运行 |
| 12 | 把安装包发给别人，并说明需安装 Python 和 FFmpeg |

按上面一步一步做即可完成「桌面打包程序」从环境准备到得到安装包、再到分发的全过程。
